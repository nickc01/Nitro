Nitro

A Powerup Helper for the Unity3D Game Engine

By Nicholas Cullen

Github Documentation Download SIP Brief

What is it?

Nitro is a library for Unity that makes it trivial to add powerups to your game.

Whether it be for a racing game, an RPG, or a battle royal, Nitro has your back.

Creating a powerup is as simple as overriding a single class and adding it into a GameObject

See Quick Start Guide 
//Example Powerup
using Nitro;
using UnityEngine;

public class SimplePowerup : Powerup
{
                public override void DoAction()
    {
        Debug.Log("Powerup Executed!");
    }
}

Features

Nitro includes many tools and components that make it easier to integrate powerups into your game, such as:

  • Revertable Variables that make it easy to adjust variables like velocity
  • Collectors that can store and execute powerups
  • Collidables that make it easier to manage collisions

Combinable Powerups

Nitro also includes the ability to create combinable powerups, which allow multiple powerups to be combined into a single powerup with enhanced effects

For example, you could combine a fire powerup and a water powerup so that a puddle spawns with fire on it

Project Updates

1.0 - Initial Demo

May 15 2021

  • Finished small demo demonstrating the Nitro Library
  • The Nitro Library can now allow for the easy creation of powerups and combinable powerups
  • The Nitro Library has the Collidable and Collidable2D components for keeping track of collided objects
  • The Library has Nitro Variables that can keep track of changes to the variable, and revert them at a later time
  • A series of Github wiki pages have been made to document how to use the library's features
  • A Primary/Auxiliary model has been created in order for powerups to be combined

2.0 - Small Game Release

Feb 13 2023

  • Improved the demo into a small game. The demo now has a main menu, music, sfx, multiplayer, and a procedurally generated road
  • The demo now utilizes online multiplayer. This is to help further demonstrate the flexibility of the Nitro Library
  • Reworked the NitroVar class into the RevertableVar class. The class now utilizes a GenericMath class to make it easier to modify the variable
  • The Collidable and Collidable2D classes are now sealed components for greater flexibility
  • The Combinable Powerup System no longer uses a Primary/Auxiliary model for combining powerups
  • A new powerup chain system has been introduced that allows multiple powerups to be combined. The collected powerups are sorted by priority, with the highest priority powerup going first and the lowest priority powerup going last. The first powerup in the chain is given a function delegate that can trigger the next powerup in the chain, and so on until all powerups have been executed. This system is more flexible than the previous Primary/Auxiliary model
